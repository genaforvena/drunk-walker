## 🧠 Why This Happens

According to `THE_TRAVERSAL_PROBLEM.md` section 2 (**Dead Reckoning & The "Drift"**):

> *"We turn the camera by holding 'Left' for a few hundred milliseconds. The Problem: Because of browser lag or how Google 'smooths' the camera move, holding the key for 600ms doesn't always mean a perfect 60° turn."*

**The Prediction Math is Broken:**

In `traversal.js`, the `predictNextLocation()` function uses the **internal orientation** to predict where the next bubble should be:

```javascript
const testOrientation = normalizeAngle(orientation + angle);
const testLocation = predictNextLocation(currentLocation, testOrientation);
```

But if `orientation` is wrong by 175°, the algorithm is:
- Scanning directions that **don't exist** in Google's actual view
- Missing the actual connected nodes because they're in a different direction than predicted
- Pressing "ArrowUp" in directions where **no node exists**

## 🛠️ Proposed Fixes

### Fix 1: Orientation Recalibration (CRITICAL)

Extract the actual yaw from the Google URL and **recalibrate** the internal orientation on every step:

```javascript
// In engine.js, after each URL change:
function extractYawFromUrl(url) {
  const match = url.match(/yaw%3D([0-9.]+)/);
  return match ? parseFloat(match[1]) : null;
}

// Recalibrate orientation when URL changes
const actualYaw = extractYawFromUrl(newUrl);
if (actualYaw !== null) {
  orientation = actualYaw; // Reset internal orientation to match Google
}
```

**Why this works:** The URL contains Google's **ground truth** yaw value. By syncing to it, the prediction math stays accurate.

---

### Fix 2: Early Loop Detection

Instead of waiting for `stuckCount >= 3`, detect when we're **returning to recently visited locations**:

```javascript
// Check if current location is in recent breadcrumbs
const isLooping = breadcrumbs.slice(-5).includes(currentLocation);
if (isLooping) {
  // Trigger early escape behavior
  return { turn: true, angle: 180 }; // Snap back immediately
}
```

---

### Fix 3: Adaptive Search Pattern

When stuck for too long (>10 ticks), expand the search beyond 60° increments:

```javascript
if (stuckCount >= 10) {
  // Fine-grained search: every 30° instead of 60°
  lastSearchAngle = (lastSearchAngle + 30) % 360;
}
```

**Why this helps:** Smaller angular steps reduce the chance of "jumping over" narrow exits.

---

### Fix 4: Random Escape (Fallback)

When truly desperate (stuck > 20 ticks), add randomness:

```javascript
if (stuckCount >= 20) {
  const randomAngle = Math.floor(Math.random() * 360);
  return { turn: true, angle: randomAngle };
}
```

**Why this works:** Breaks the deterministic pattern that might be missing all exits due to drift.

---

### Fix 5: Breadcrumb-Weighted Exploration

Increase the penalty for returning to very recent locations:

```javascript
// In the scoring function
breadcrumbs.forEach((bc, index) => {
  // Recent breadcrumbs (lower index) get higher penalty
  if (bc === testLocation) breadcrumbPenalty += (100 - index * 5);
});
```
